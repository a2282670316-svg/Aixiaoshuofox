# Changelog

## v0.7.0 - 2026-07-12

### 正确性与安全

- AI 请求增加阶段参数校验、用户级频率限制、并发限制、浏览器取消传递和明确的 499 取消响应。
- 修正前台与后台调用预算统计：每次模型尝试在发送前占用次数，失败和取消请求同样计入。
- 自动清理过旧的后台响应、Webhook 事件和多余运行历史。

### 一致性闭环

- 一致性问题增加稳定指纹，复审时优先识别同一问题，降低“修复后又出现新编号”的噪声。
- AI 修复从整章重写改为受限局部补丁，并验证原文唯一匹配、修改数量和改动比例。
- 事实账本增加作者设定、正文证据、AI 已核验和推断四级可信度，续写上下文只使用可验证事实。

### 持久化与性能

- IndexedDB 和 D1 的大工作区启用 Gzip 压缩，同时兼容未压缩旧数据。
- 后台任务先轮询轻量摘要，只有版本变化或任务结束时才下载完整作品。
- 浏览器保存增加节流和离开页面补写，降低连续写作时的重复序列化与写盘。

### 工程质量

- 删除未引用的旧蓝图提示、恢复函数、示例 API 和脚手架图片。
- 新增统一 `npm run check`、生产依赖安全检查、GitHub CI 和带校验和的 Release 源码包流程。

## v0.5.0 - 2026-07-11

### 长篇写作安全与容量

- 统一分阶段模型 Token 上限，最高支持 65,536 Token，并修正调用次数和 Token 预算统计。
- 作品与备份数据迁移到 IndexedDB，自动兼容旧 localStorage 数据。
- 限制章节版本、已解决问题和任务日志的长期容量，避免长篇写作数据无限膨胀。
- 云端检查点增加版本冲突检测和安全合并，保护浏览器中更新的人工正文。

### 证据化写作闭环

- 章纲执行率改为基于正文原句的证据验证，不再仅检查字段是否存在。
- 伏笔埋设、推进和回收需要对应正文引用，并进入独立伏笔账本。
- 增加人物位置、身体、情绪、知识、持有物和目标追踪，检测知识倒退和无过渡地点变化。
- 全书错误修复队列按上游根因和后续影响组织，先修根因，再修派生问题。

### 阶段恢复与任务中心

- 新增阶段恢复中心，可查看蓝图、正文分段、记忆、审校和修复检查点。
- 支持从指定蓝图阶段、章节和正文分段继续写作。
- 页面重新打开时自动识别未完成任务，避免任务永远显示为运行中。
- 拆分独立任务中心组件，精简重复的浏览器和云端续写入口。

### 质量与安全

- 修正无 API Key 的 Ollama 等本地模型连接状态。
- 为创作助手补充无障碍标签，并确认设置页可正常滚动。
- TypeScript、ESLint、生产构建和 50 项自动化测试全部通过。
- 生产依赖安全检查为 0 个已知漏洞。
