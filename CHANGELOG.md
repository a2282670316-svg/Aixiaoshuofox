# Changelog

## v0.5.0 - 2026-07-11

### 长篇写作安全与容量

- 统一分阶段模型 Token 上限，最高支持 65,536 Token，并修正调用次数和 Token 预算统计。
- 作品与备份数据迁移到 IndexedDB，自动兼容旧 localStorage 数据。
- 限制章节版本、已解决问题和任务日志的长期容量，避免长篇写作数据无限膨胀。
- 云端检查点增加版本冲突检测和安全合并，保护浏览器中更新的人工正文。

### 证据化写作闭环

- 章纲执行率改为基于正文原句的证据验证，不再仅检查字段是否存在。
- 伏笔埋设、推进和回收需要对应正文引用，并进入独立伏笔账本。
- 增加人物位置、身体、情绪、知识、持有物和目标追踪，检测知识倒退和无过渡地点变化。
- 全书错误修复队列按上游根因和后续影响组织，先修根因，再修派生问题。

### 阶段恢复与任务中心

- 新增阶段恢复中心，可查看蓝图、正文分段、记忆、审校和修复检查点。
- 支持从指定蓝图阶段、章节和正文分段继续写作。
- 页面重新打开时自动识别未完成任务，避免任务永远显示为运行中。
- 拆分独立任务中心组件，精简重复的浏览器和云端续写入口。

### 质量与安全

- 修正无 API Key 的 Ollama 等本地模型连接状态。
- 为创作助手补充无障碍标签，并确认设置页可正常滚动。
- TypeScript、ESLint、生产构建和 50 项自动化测试全部通过。
- 生产依赖安全检查为 0 个已知漏洞。
